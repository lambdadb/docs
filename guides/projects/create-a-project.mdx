---
title: Create a project
description: Create a new LambdaDB project via the admin API. Set the project name, rate limits, snapshot retention, and resource quotas with a single request.
---

This page describes how to create a project.

```bash
LAMBDADB_ADMIN_API_KEY="ADMIN_API_KEY"

curl -i -X POST "https://{baseUrl}/projects" \
  -H "content-type: application/json" \
  -H "x-api-key: $LAMBDADB_ADMIN_API_KEY" \
  -d '{
        "projectName": "example-project",
        "rateLimit": 10.0,
        "snapshotRetentionInDays": 30,
        "quotaEnabled": false
      }'
```

The response will look like this:

```json
{
  "project": {
    "projectId": "<string>",
    "projectName": "example-project",
    "apiKey": "<string>",
    "rateLimit": 10.0,
    "snapshotRetentionInDays": 30,
    "status": "CREATING",
    "quotaEnabled": false
  }
}
```

To create a project with quota enabled:

```bash
LAMBDADB_ADMIN_API_KEY="ADMIN_API_KEY"

curl -i -X POST "https://{baseUrl}/projects" \
  -H "content-type: application/json" \
  -H "x-api-key: $LAMBDADB_ADMIN_API_KEY" \
  -d '{
        "projectName": "example-project-with-quota",
        "rateLimit": 10.0,
        "snapshotRetentionInDays": 30,
        "quotaEnabled": true,
        "quota": {
          "period": "monthly",
          "read": { "limit": 1000000 },
          "write": { "limit": 500000 },
          "storage": { "limit": 10737418240 }
        }
      }'
```

<Note>
When `quotaEnabled` is `true`, `quota` is required.
`quota.period` must be one of `daily`, `weekly`, `monthly`.
Also, all of `quota.read.limit`, `quota.write.limit`, and `quota.storage.limit` are required.
</Note>
