title: Manage projects

This page explains how to manage projects.

## List projects

Use this operation to get a complete description of all projects in an organization:

```bash
LAMBDADB_ADMIN_API_KEY="ADMIN_API_KEY"

curl -i -X GET "https://{baseUrl}/projects" \
  -H "content-type: application/json" \
  -H "x-api-key: $LAMBDADB_ADMIN_API_KEY"
```

The response will look like this:

```json
{
  "projects": [
    {
      "projectId": "<string>",
      "projectName": "<string>",
      "apiKey": "<string>",
      "rateLimit": 10.0,
      "snapshotRetentionInDays": 30,
      "status": "CREATING",
      "quotaEnabled": false
    }
  ]
}
```

## Describe a project

Use the describe endpoint to get a complete description of a specific project:

```bash
LAMBDADB_ADMIN_API_KEY="ADMIN_API_KEY"

curl -i -X GET "https://{baseUrl}/projects/{projectName}" \
  -H "content-type: application/json" \
  -H "x-api-key: $LAMBDADB_ADMIN_API_KEY"
```

The response will look like this:

```json
{
  "project": {
    "projectId": "<string>",
    "projectName": "<string>",
    "apiKey": "<string>",
    "rateLimit": 10.0,
    "snapshotRetentionInDays": 30,
    "status": "ACTIVE",
    "quotaEnabled": false
  }
}
```

## Configure a project

Use the update endpoint to change the project configuration (for example `rateLimit`, `quotaEnabled`, `quota`):

```bash
LAMBDADB_ADMIN_API_KEY="ADMIN_API_KEY"

curl -i -X PATCH "https://{baseUrl}/projects/{projectName}" \
  -H "content-type: application/json" \
  -H "x-api-key: $LAMBDADB_ADMIN_API_KEY" \
  -d '{
    "rateLimit": 20.0,
    "quotaEnabled": true,
    "quota": {
      "period": "monthly",
      "read": { "limit": 1000000 },
      "write": { "limit": 500000 },
      "storage": { "limit": 10737418240 }
    }
  }'
```

The response will look like this:

```json
{
  "project": {
    "projectId": "<string>",
    "projectName": "<string>",
    "apiKey": "<string>",
    "rateLimit": 20.0,
    "snapshotRetentionInDays": 30,
    "status": "ACTIVE",
    "quotaEnabled": true,
    "quota": {
      "period": "monthly",
      "read": { "limit": 1000000 },
      "write": { "limit": 500000 },
      "storage": { "limit": 10737418240 }
    }
  }
}
```

<Note>
`PATCH /projects/{projectName}` requires at least one of `rateLimit` or `quotaEnabled`.
If `quotaEnabled` is `true`, `quota` is required. If `quotaEnabled` is `false` or omitted, `quota` must be `null`/omitted.
When `quotaEnabled` is `true`, `quota.period` must be one of `daily`, `weekly`, `monthly`.
Also, all of `quota.read.limit`, `quota.write.limit`, and `quota.storage.limit` are required.
</Note>

## Delete a project

Use this operation to delete a project and all of its associated resources:

```bash
LAMBDADB_ADMIN_API_KEY="ADMIN_API_KEY"

curl -i -X DELETE "https://{baseUrl}/projects/{projectName}" \
  -H "content-type: application/json" \
  -H "x-api-key: $LAMBDADB_ADMIN_API_KEY"
```

<Warning>
Deleting a project causes deleting all the collections in the project.
</Warning>
